啥是 Git？
	分布式版本控制系统，做协同开发。
Git的诞生
	Linux linus
分布式和集中式
Git安装:设置名字和邮箱	
	$ git config --global user.name "youname"

	$ git config --global user.email "emall@qq.com"
创建版本库
	创建空目录，作为版本库
	初始化版本库
		1.进入版本库所在的目录
		2.执行初始化命令
		Git init 

管理文本，代码，网页
把文件添加到版本库
	1.创建文件夹
	2.在仓库里查看状态
		git status
	3.把文件添加盗版本库
		git add 文件名
	4.把文件提交给版本库
		git commit -m "添加了一个文件test.txt"
	5.查看文件每一个版本
		git log

时光穿梭
	回到以前版本
	git reset --hard 版本
	版本: HEAD:代表当前版本
		  HEAD^:代表上个版本
		  HEAD^^:上2个版本
		  HEAD~10:上10个版本
	git reset --hard HEAD^
	后悔药(后退错了方可服用)：
		git reflog:查看历史所有版本信息，找到版本号
		git reset --hard 版本号 
		
修改撤销:
	git checkout -- 文件名

清除暂存区的文件：
	git reset HEAD 文件名 

删除文件：
	不想删: 就恢复出来	git checkout -- 文件名	   
	“真”想删: 把删除的状态提交到版本库，就没了
	真的真的想删: 删除隐藏文件 .git

Github
	
将本机的SSH添加到Github
	1.在本地创建SSH key
		ssh-keygen -t rsa -C "邮箱"
	2.找到钥匙锁在的目录
		c://User/web/.ssh/
	3.复制id_rsa.pub内容到Github
	4.测试连接
		ssh -T git@github.com

本地仓库上传文件到GitHub
	1.连接远端仓库s
	    查看连接详细  git remote -v
	    git remote add origin  钥匙
	   例： git remote add origin   git@github.com:gouhuiyang/GitTest.git
	2.将文件推送到远端仓库
	    查看分支状态
		git pull
	    将远程origin的master拉到本地，与本地分支master结合:
	    	git pull origin master
	    然后方可上传文件:
	    	git push -u origin master（写的是分支的)
	3.如果远程仓库已存在就先删除
	   查看仓库详细地址 git remote -v
	   删除仓库 git remote rm origin 
把远程仓库克隆到本地
	   git clone 地址
协同开发：

参加开源项目
	1.先找到你要参加的开源项目，fork到自己的github仓库
	2.把自己的fork的项目clone到本地
	3.修改项目
	4.把修改好的项目push到自己的github库
	5.把自己修改好的pull给作者
